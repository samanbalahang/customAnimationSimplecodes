Each instance of SVG content in our HTML needs its own SVG document (opening/closing SVG tags) and if the text needs to wrap, make a separate SVG doc for each wrappable bit of text and put it inside a span so it flows inline; don't forget to add aria-labels to each doc for accessiblity. As for responsiveness, each document needs its own relative width and height set using CSS, not the tag attributes.

The two most defining constraints are that an SVG document has to have its width and height set explicitly or it doesn't paint and all our SVG content must adapt responsively. Although you need a vector design app to develop these gradients and get the initial SVG markup, once you plug that into your HTML, stay in the browser for all your subsequent creating and recalibrating. Set or adjust the SVG tag viewbox first, then set the relative widths and heights in the CSS accordingly. If the text needs to wrap, the task becomes more tedious but is essentially the same.
